从 2D 像素 到 3D 物理 的桥梁

affine_transform.py
Python 
import numpy as np

# 标定矩阵 M (由 calibrate.py 获得)
M = np.array([
    [ 0.52, -0.04, -120.5],
    [-0.03,  0.55,   45.2],
    [ 0.00,  0.00,    1.00]
])

def pixel_to_arm(u, v):
    """将像素坐标(u,v)转换为机械臂坐标(x,y)"""
    # 1. 构建齐次坐标向量
    pixel_point = np.array([u, v, 1])
    
    # 2. 矩阵乘法: Arm = M @ Pixel
    arm_point = np.dot(M, pixel_point)
    
    return round(arm_point[0], 2), round(arm_point[1], 2)

# --- 运行测试 ---
u, v = 552, 347
x, y = pixel_to_arm(u, v)
print(f"Pixel({u}, {v}) -> Robot({x}, {y})")

核心知识点
具身智能 (Embodied AI)

将 AI 大脑赋予物理实体，使其能通过“感知-行动”闭环与现实世界交互，而不仅仅是处理文本。

ReAct 范式

Reasoning + Acting。智能体在执行动作前先进行“思考”，根据观察结果动态调整下一步计划。

VLM 视觉语言模型

Vision-Language Model。不仅能“看”到像素，还能“理解”图像语义，如识别“红色的苹果”。

工具学习 (Tool Use)

LLM 本身无法操作硬件，通过封装 Python 函数（如 `grab_object`）作为“工具”，让 LLM 学会调用。

仿射变换 (Affine)

解决 2D 图像坐标 (u,v) 到 3D 机械臂坐标 (x,y) 的映射难题，消除透视畸变。

实验目标

本实验旨在通过软硬件结合，实现一个基于 大语言模型 (LLM) 和 视觉语言模型 (VLM) 的智能机械臂 Agent。

  • 1. 理解原理：掌握六轴机械臂硬件构成及基于 LLM 的 Agent 控制逻辑。
  • 2. 剖析算法：深入分析 VLM 原理及图像空间到物理空间的坐标转换 (仿射变换)。
  • 3. 编程实践：编写 Python 代码实现“感知-决策-执行”闭环。

4. 核心工具开发 (tools.py)

Agent 的能力取决于工具的实现。我们需要在 tools.py 中完善以下三个函数：

逻辑流程：

  1. 参数校验： 检查 `area_description` 是 "grabbing_area" 还是 "placement_area"。
  2. 机械臂移动：
    • 如果是抓取区，移动到预设坐标 A (高处俯视)。
    • 如果是放置区，移动到预设坐标 B。
  3. 图像采集： OpenCV 读取摄像头 `cv2.VideoCapture(0)`。
  4. 图像处理： 关键！ 进行 Gamma 校正 (gamma=2.1) 以提高亮度，保存为 `captured_image.jpg`。

逻辑流程：

  1. VLM 推理： 读取图片，调用 Qwen-VL API，提示词 "一个{object_name}"，获取像素坐标 `[x1, y1, x2, y2]`。
  2. 计算中心点： `center_x = (x1+x2)/2`, `center_y = (y1+y2)/2`。
  3. 坐标转换： 调用 `eyeonhand.pixel_to_arm(center)` 将像素转为机械臂坐标。
  4. 应用偏移： 读取 `config.json` 中的 `x_offset`, `y_offset` 进行微调。
  5. 执行动作：
    open_gripper() -> 移动到上方(z=200) -> 下放(z=120) -> close_gripper() -> 抬起(z=200)

参数多态处理：

  • 情况 A：传入坐标列表 (如 `[100, -200]`)
    -> 直接调用机械臂移动接口。
  • 情况 B：传入物体名称 (如 `"trash bin"`)
    -> 1. 自动触发放置区拍照逻辑。
    -> 2. 调用 VLM 识别垃圾桶坐标。
    -> 3. 调用 `pixel_to_arm_place` 转换坐标。
    -> 4. 移动并在目标上方释放物体。

5. 关键步骤：手眼标定

这一步决定了机械臂是“抓到物体”还是“抓空气”。必须非常耐心。

标定原理 (Affine Transformation)

我们通过采集 4 组 (像素坐标 u,v - 机械臂坐标 x,y) 的对应点，利用最小二乘法解算出变换矩阵。

Step 1: 物理打点 (x, y)
  • 控制机械臂尖端，依次触碰工作纸上的 4 个角。
  • 记录下每次触碰时机械臂的 X, Y 坐标。
  • 填入 `config.json` 的 "points_arm"。
Step 2: 视觉采样 (u, v)
  • 机械臂回到拍照姿态，保持不动拍照。
  • 在照片上用工具(如画图)找 4 个角的像素坐标。
  • 填入 `config.json` 的 "points_pixel"。
Config.json 示例
{
    "points_pixel": [[320,220], [590,430], [72,31], [590,26]], 
    "points_arm":   [[210,0],   [140,-80], [280,80], [280,-80]],
    ...
}

最后运行标定脚本生成矩阵：python3 eyeonhand.py

6. 进阶：放置区标定

如果要实现 move to trash bin (语义移动)，仅标定抓取区是不够的，因为放置区在摄像头的另一侧，透视关系不同。

操作流程：
  1. 确定拍照位： 调整机械臂到放置区上方，确定一个新的拍照坐标，填入 `take_photo` 的 else 分支。
  2. 放置区打点： 同样的方法，获取放置区 4 个角的机械臂坐标和像素坐标。
  3. 更新配置： 将数据填入 `config.json` 的 points_pixel_place 和 points_arm_place。
  4. 代码适配： 确保 `move_to` 函数在处理物体名称时，调用的是 `eyeonhand.pixel_to_arm_place()` 函数。
提示： 为了方便对齐，可以将 Z 轴降至 110 (离纸面近一点) 来进行物理打点。
